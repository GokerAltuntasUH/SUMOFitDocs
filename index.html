domain

domain_services:
Contains helper functions to modify/process data in the Network class element

The domain_services sub-module includes functions that modify the network. This approach helps keep the Network class definition clean and conserve its data-oriented role.

Functions

  • move_node(network: Network, node_guid: str, move_vector: tuple[float, float])
    Moves the Node class instances by the given vector and automatically updates the Loop class and EdgeChain class instances the node is associated with; returns None.
    Expected to be triggered by the move_node() method of the SUMOFitEnv class.
  • process_node_spatial(network: Network, node_guid: str, spatial_state: RealisationFormEnumeration)
    Upon the agen't decision to change the realisation form of a Node class instance, if the action is valid, it automatically updates the RealisationForm of the Loop class instances the node is associated with in the Network. Updating FlowSet and Network class instances in parallel is handled by the SUMOFitEnv class. The function returns None.
    Expected to be triggered by the spatial_change() method of the SUMOFitEnv class.

🌐 Network Class:
The collection of all the infrastructure-related elements.

The Network class can be taken as the Infrastructure equivalent of the FlowSet class. The class is designed to store all Network-related elements in memory at runtime, including the training phase. Network class is defined in a data-oriented approach. Most methods are organised in domain_services and queries sub-modules. However, basic, short-code methods remain in the Network class.

Properties

  • metadata
    A metadata class instance associated with the Network. The NetworkMetadata class has not yet been explained in the document. Practically, it contains information about:
    • Export date from the Civil3D model
    • NetExport version (a complementary project) that exported the model info
    • Source of the model, defaults to "Civil3D"
    • A unique identifier of the Network element
    • HotMap code (a complementary project) that identifies the location
    • BIM file version addressing the revisions in the Existing Condition modelling
    • Whether the default flows by the SUMOFit would be applied
    • Total number of primary nodes in the network
    • Total number of transition nodes in the network
    • Total number of auxiliary nodes in the network
    • Total number of pole nodes in the network
    • Total number of edges in the network
  • node_manager
    The NodeManager class instance that is associated with the Network class instance.
  • edge_manager
    The EdgeManager class instance that is associated with the Network class instance.
  • chain_manager
    The ChainManager class instance that is associated with the Network class instance.
  • _archive
    An archived status of the Network, allowing the SUMOFitEnv to fast-reset the network back into its initial state without.
  • _validated
    Whether the Network is valid by means of
    • All the edges have a valid starting and ending node
    • All the node coordinates are within the limits of Normalisatin Range

Methods

  • Parsers, Constructors and Serialisers
    • to_dict()
      Returns a dictionary representation of the Network class instance, allowing archiving and restoring.
    • archive_network(json_data: Dict[str, Any])
      Checks whether the archived version of the Network exists in the _archive property. If it does not, constructs one using the to_dict() method.
    • from_dict(data: Dict[str, Any], source: str):
      Constructs all managers using their from_json_data() methods from the given source. The source argument should be either "civil3d" or "archive"; default to "archive".
    • restore_from_archive(archive: Optional[Dict[str, Any]])
      Calls from_dict() method with the "archive" source code and restores the network from the given archive, ensuring the archive is not mutated. If no archive is given, the network is reconstructed by the _archive property.
  • Access Methods
    • get_node(NodeGuid: str, destination: str):
      Returns the Node class element with the given NodeGuid.
    • get_edge(edge_guid: str):
      Returns the Edge class element with the given EdgeGuid.
    • get_loop(loop_id: str):
      Returns the Loop class element with the given EdgeGuid.
  • Processors
    • validate_network(flow_id: str):
      Enables SUMOFitEnv instances to validate the network before pre-processing.
  • Registering and Indexing
    • add_loop(flow: FlowElement): A loop registration method is defined to allow FlowSet class instances to discover and implement the Loop during FlowElement creation without providing those instances with direct access to the ChainManager class.
      Expected to be triggered by FlowSet class's _calculate_routes method, where "realisation_flag" argument is "True".
🔀 FlowSet Class:
Collection of the FlowElement class instances travels through the Network


The FlowSet class can be taken as the demand equivalent of the Network class. A FlowSet class instance is generated from the network, automatically calculating all the possible flows and discovering loops with the potential to act as a Roundabout. The major architectural difference is that the FlowSet class does not yet have entity-specific managers, such as the NodeManager and ChainManager classes.

Properties

  • flows
    A dictionary of FlowElement class instances, indexed by the FlowId
  • flows_by_edge
    A dictionary of lists of flow elements indexed by the EdgeGuid. The dictionary is used to quickly access the FlowElement class instance that travels along the given edge.
  • default_vehicle
    A default vehicle type is applied to FlowElement instances within the FlowSet.
  • noo_loop_duplicates
    A boolean value indicating whether a FlowElement that traverses only non-loop roads should override and remove FlowElement instances that include at least one loop. This primarily influences the handling of roundabouts when both routes share the same origin–destination key.
  • od_index
    A dictionary of FlowElement lists indexed by the origin-destination key.
  • flow_count
    An integer corresponds to the number of FlowElement instances in the FlowSet instance.

Methods

  • Parsers, Constructors and Serialisers
    • from_network(network: Network, realisation_flag: bool, no_loop_duplicates: bool, default_vehicle: Optional[VehicleType], default_flowSchema: Optional[FlowSchema]):
      Creates the FlowSet instance from the given Network class instance.
      Arguments:
      • network
      • realisation_flag
        Whether realisation forms of the netowrk elements should be identified during the creation of the FlowSet. If yes, identified loops are registered with the Network class instance's ChainManager.
      • no_loop_duplicates
        Whether flows that travel through a loop should be eliminated, where an alternative route can be found.
      • default_vehicle
        The default vehicle type is to be applied to the FlowElement instances created.
      • default_flowSchema
        A template FlowElement to be used during the creation of the FlowElement instances in the FlowSet.
    • from_dict(json_data: Dict[str, Any])
      Creates a FlowSet instance from the given archive and returns the created FlowSet element
    • restore_from_archive(archive: Optional[Dict[str, Any]])
      Restores the FlowSet to its initial state by passing the Archive to the from_dict() method. If no archive is provided, it will use a pre-built archive created during the creation of the FlowSet. Calling the from_dict() method through the restore_from_archive() method is a legacy architecture inherited from the Network class. This practice may be redundant in later versions.
    • archive_flowset()
      Archives the current state of the FlowSet, enabling recreation of the FlowSet together with the FlowElement instances.
  • Access Methods
    • get_by_od(origin: str, destination: str):
      Returns a list of FlowElement instances whose origin edge and the target edge match the given.
  • Processors
    • process_loop_spatial(flow_id: str):
      Given a Loop element, relevant FlowElement instances decide on ommitting and unommitting the Edge instances in the Loop.
  • Registering and Indexing
    • add_flow(flow: FlowElement): Registers the FlowElement class instance with the correct indexing.
    • remove_flow(flow_id: str): Removes the FlowElement class instance and updates all indices.

🔗 EdgeChain:
Collection of Edges separated only by Auxiliary Nodes.


In SUMO, Edges may have a polyline geometry. An "Edge" in SUMOFit represents a single segment, while an EdgeChain represents the entire Edge in SUMO. When writing the "Edges.XML" (See Scenario Builder) Edge elements of SUMOFit are ignored, and their collective forms, EdgeChain elements, are translated into SUMO edges.

Properties

  • RegionGuid
    Civil3D Region Id that the instance of the EdgeChain refers to.
  • RegionAssemblyId
    Civil3D Assembly Id applied to the Civil3D region.
  • Edges
    A list of Edge class elements, always ordered by the Civil3D alignment station value that the Edge class element overlaps.
  • _start_stations
    Civil3D alignment station that the Civil3D region corresponding to the EdgeChain element starts.
  • LocationOffset
    Calculated by ChainManager class in consideration of the total road width and the Civil3D assembly insertion line.
  • shape
    Calculated by ChainManager class using the LocationOffset property of the EdgeChain class instance. The value is directly projected onto SUMO to ensure the physical centre of the road aligns with the SUMO Edge's insertion reference.

Methods

  • Parsers, Constructors and Serialisers
    • to_archive_dict():
      Returns a dictionary representation of the EdgeChain class instance, allowing archiving and restoring.
    • from_archive_dict(data: Dict[str, Any]):
      Constructs the EdgeChain class instance from the Network class instance's internal archive.
    • add_edge(edge: Edge):
      Inserts a new edge into the EdgeChain, conserving the order by the Civil3D alignment station.


🔢 Enumerators:
Helps implement string-type properties by assigning integers


Helps implement string-type properties by assigning integers

Elements:

  • Realisation Form Enumeration
    The state that defines the realisation form of the abstract elements
    • Free Flow:
      Profiles are extruded along abstract lines
    • Roundabout:
      Profiles are extruded along abstract lines. Edge priorities are set to form a roundabout
    • Super Node:
      The entire loop geometry collapses into one super node

  • Node Type Enumeration
    Helps categorise the Node elements
    • Primary Node:
      Represents the intersections of abstract, linear elements, as well as the intersection of corridors
    • Transition Node:
      Represents where the abstract line features are consistent, but the corridor properties change (i.e. an extra lane)
    • Auxiliary Node:
      Represents where the corridor properties are consistent, but the abstract line features change (i.e. the beginning of a curvature). Auxiliary nodes are not explicitly present in SUMO, but form the "SHAPE" property of a SUMO edge. In SUMOFit, two or more SUMOFit edges that bond with auxiliary nodes create an EdgeChain A SUMOFit EdgeChain renders as an Edge in SUMO
    • Pole Node:
      Represents the endpoints of the network. Also represents the Origins and the Destinations of Flow Elements. The position of a Pole Node can not be modified

  • Edge Type Enumeration
    Helps categorise the Edge Elements. Edge types do not yet affect the simulation. In the future, we expect to include "spiral" type of edges in addition to "Arc"
    • Connecting:
      Restores the topographic structure in which two primary nodes are located adjacent to each other. They are not represented in Civil3D and are produced during data extraction, where a junction's geometry is flushed with another's.
    • Line:
      Represents a direct connection between two nodes
    • Arc:
      Represents a curve between two nodes. Using Arc-type edges helps define curvatures without segmenting the road geometry.

  • Conflict Type Enumeration
    Replicates SUMO-defined conflict types
    • Further information can be found at https://sumo.dlr.de/docs/Simulation/Output/SSM_Device.html


➡️ Flow Element Class:
A class that is to align with SUMO's internal flow mechanism


Represents a series of EdgeChains to pass through when travelling from specific origins to destinations.

Properties

  • vehicleTypeId
    Refers to the type of vehicle in the series that will follow the flow during the simulation.
  • FlowId
    Civil3D Assembly Id applied to the Civil3D region.
  • edges
    An ordered series of Edge class instance guids that the vehicles in the flow will follow.
  • begin
    The simulation time at which the flow begins.
  • end
    The simulation time at which the flow exterminates.
  • vehsPerHour
    Total number of vehicles to be deployed into the simulation between the start and the end
  • through_loops
    Whether the flow goes through a loop definition. The property is set during creation to help avoid duplicating "Origin"-"Destination" pairs. Where two flow elements share the identical origin and destination keys, the one that goes through loops is eliminated. In doing so, the vehicles prefer diagonal connections to roundabouts and highway junctions. Currently, this operation uses a costly method explained in the FlowSet class class.
  • ommitted_edges
    Edges that are part of a loop need to be omitted when writing the XML if the Realisation Form is a "SuperNode". Property keeps a list of omitted edges, enabling switching between the Roundabout and Freeflow states.

🎮 SUMOFitEnv Class:
An application of the Gymnasium environment over the SUMO network


SUMOFitEnv class instances create Gymnasium environments using the given Network class instance. The instance is responsible for modifying the network based on the agent's decisions and calculating rewards by running the simulation.

Properties

  • Environment-wide Properties:
    The following properties are conserved between the episodes
    • base_network
      The Network class instance to be modified and evaluated by the SUMOFitEnv element.
    • base_flowset
      The FlowSet class instance represents the auto-generated demand elements
    • test_name
      A string value is given at each run. Simulation files, results and ML weights are stored using the test_name property of the SUMOFitEnv. On a project-wide basis, a test name is automatically assigned using a timestamp.
    • sumo_seed
      An integer value controls the randomness in SUMO.
    • _scenario_builder
      A SUMOSCenarioBuilder class instance that creates SUMO-specific files at each step.
    • score_dict
      A dictionary maps the metrics to be collected from SUMO, along with their weights used to calculate the environment score at each step.
    • N_MAX
      The maximum number of nodes that can be present in the Network class instance, including additional nodes to be created by the agent.
    • E_MAX
      The maximum number of edges that can be present in the Network class instance, including additional nodes to be created by the agent.
    • Fgeom
      Number of geometric features of nodes, currently limited to X and Y coordinates.
    • max_episode_steps
      Number of steps to terminate an episode, if any other threshold is not met.
    • max_move
      Maximum length of the move vector that can be applied to a node by the agent.
    • observation_space
      A dictionary element to store the agent's observation at each step.
    • action_space
      A dictionary element defines the scope of actions the agent can take.
  • Construction Properties:
    The following properties are calculated before reset() and retained during training, avoiding costly recomputation after each episode. The word "construction" refers to the values calculated during the initialisation of the SUMOFitEnv using the base Network class instance.
    • construction_node_features
      Refers to the normalised values of the node features.
    • construction_node_types
      Refers to the integer values of the node types
    • construction_spatial_values
      Refers to the integer values of the RealisationFormEnumeration values
    • construction_node_mask
      Refers to the padding that indicates slots reserved for future nodes created by the agent. Node creation logic is currently under development.
    • construction_slot_to_guid
      Maps the Node slots in the SUMOFitEnv to the Node instances in the Network class instance.
    • construction_guid_to_slot
      Maps the Node instances in the Network class instance to the Node slots in the SUMOFitEnv.
    • construction_score
      Keeps the score at the initial state of the Network class instance to avoid the simulation running after each reset.
  • Step-specific properties:
    SUMOFitEnv logic updates the following properties after each step in an episode.
    • _last_obs
      A dictionary to track the latest state of the Network class instance, initialised by the:
      • construction_node_features
      • construction_node_types
      • construction_spatial_features
      • construction_node_mask
      properties of the network, and updated in each step without altering the original properties.

Methods

  • Stepping Logic
    • reset(seed: int, options: Dict[str, Any]):
      Reconstructs the Network and the FlowSet before an episode starts. As with general practice, the reset() method is always called when a SUMOFitEnv is initialised. The method avoids running the simulation. The initial score is rather calculated during the class's initialisation, only once for efficiency. The method returns the _last_obs property and additional info.
    • step(acton: Dict[str, Any]):
      Applies the action specified by the action argument, then calculates the score. The reward is calculated by the difference between the current and the previous scores. The method returns the class's _last_obs property, the calculated reward, whether the step is terminated or truncated, and any additional information.
    • move_node(action: dict[str, tuple], node_ind: int):
      Moves the node with the given node_ind by the given move vector in the action argument. The method returns a boolean indicating whether the action was successful, with a message.
      Expected to be triggered by the self's step() method.
    • spatial_change(action: dict[str, tuple], node_ind: int):
      Triggers a change in the "RealisationForm" in a Loop class, if the node with the given node_id is involved in the Loop. The method returns a boolean indicating whether the action was successful, with a message.
      Expected to be triggered by the self's step() method.
  • Simulation Logic
    • create_sumo_scenario(episode_id: int, step_id: int, overrite: bool):
      Creates the SUMO files for the given episode and given step, and returns the step-specific directory, a common file name shared between SUMO files, and the unique TraCI label to differentiate between simulations run in parallel.
      Expected to be called by the self's step() method.
    • calculate_network_score(target_dir: str, common_file_name: str, traci_label: str):
      Given the directories, triggers a SUMO simulation and returns the score for the current state of the Network class instance.
      Expected to be called by the self's step() method.
  • Helpers
    • _build_observation_tensors(nodes_dict: dict[str, Any]):
      Given the normalised features of the Node class instances constructs several Numpy data structures, including
      • node_features: Space representing the normalised node coordinates
      • node_types: Space representing the node types
      • spatial_features: Space representing the realisation form of all the nodes
      • node_mask: Space representing the nodes in the network, and padding slots completing the number of nodes to the maximum number of nodes
      • slot_to_guid: Mapping the specific cells of the node_mask to the NodeGuid values
      • guid_to_slot: Mapping the NodeGuid values to the specific cells of the node_mask

Known Issues

Logical Inefficiencies

  • Simulation results are written to disk and then parsed back into RAM, creating a massive slowdown.
  • Edge elimination logic is missing

Data

  • SUMOXml is created from scratch each time the network is modified
    • For better performance, keep a copy of the XML text and modify it using XML queries.
  • Connecting edges uses a default lane setup, which is not perfectly realistic
  • FlowSet creates loops
    • They are included in the archive at the moment of creation by a FlowSet
    • A better architecture may be required
  • Managers always return a list from a dictionary. For example, for NodeManager class, keeping a master dictionary and sets would be faster.
  • Documentation Issues
    • There is an inconsistency between introducing a new Loop and a new EdgeChain. For clarity, both should be managed by modules at the same level of hierarchy.
    • The documentation has intense cross-references. For a better reading experience, references should be hyperlinked to the relevant sections.
    • Methods and functionalities are explained in a way that makes the information flow clear and the hierarchy visible. However, class properties have not yet reached the same level of clarity. It is still unclear which property is filled by which function and when it's used.
    • The phrase "Expected to be triggered" is often preferred in the document. On the other hand, "Expected to be called" would better align with the industry standards.
